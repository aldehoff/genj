Travailler avec les Menus Contextuels

La chose la plus importante à essayer en travaillant avec GenJ est de vous servir des menus contextuels. Un clic droit (votre système peut nécessiter d'agir différemment pour activer cette action) sur l'information affichée dans l'une des fenêtres de visualisation vous permettra de voir différents types d'actions possibles.

Le menu contextuel est toujours structuré de la même manière : en premier lieu, il affiche un sous menu lié à ce sur quoi, vous avez cliqué, et éventuellement il peut vous afficher d'autres sous menus qui élargissent les possibilités en fonction du contexte.

Dans l'exemple qui suit, on montre le contexte pour la propriété Nom (INDI:NAME). Le premier sous menu concerne la propriété elle-même et les actions possibles qui y sont liées. Le second sous menu concerne l'entité (ici un individu) à laquelle cette propriété appartient (c'est le Nom d'un Individu). Le troisième sous menu concerne le fichier Gedcom auquel cette entité appartient.

Le sous menu pour l'Individu 'I008' présente les actions suivantes :
  • Créer un enfant (Add Child) - on va créer un enfant (et une famille si nécessaire) pour l'individu I008.
  • Créer un(e) Epoux(se) (Add Spouse) - on va créer un(e) époux(se) (et une famille si nécessaire) pour l'individu I008.
  • Créer un(e) Frère(Soeur) (Add Sibling) - on va créer un(e) Frère(Soeur) (et des parents si nécessaire) pour l'individu I008.
  • Créer un lieu de stockage (Add Repository) - on va créer un enregistrement pour un lieu de stockage externe contenant des informations pour l'individu I008.
  • Ajout d'un Objet Multimédia (Add Multimedia Object) - on va créer un objet contenant des informations de type binaire.
  • Ajout d'une Note (Add Note) - création d'une note décrivant l'individu I008.
  • Ajout d'un Fournisseur d'Information (Add Submitter) - on va créer la personne ayant fourni les informations à propos de l'individu I008.
  • Ajout d'une Source (Add Source) - Création d'une source d'information concernant l'individu I008.
  • Ajout d'une Association (Add Association) - Création d'une association avec un autre Individu.
  • Effacer (Delete) - Permet la suppression de l'Entité I008.

L'exemple suivant montre à nouveau un menu contextuel pour une propriété, mais cette fois les actions pour la propriété elle-même sont présentées (premier sous menu) :

Voici les actions qui vous sont offertes :
  • Ajout d'une Note (Add Note) - cette note va pouvoir contenir des informations spécifiques à cette naissance.
  • Ajout d'une Source (Add Source) - Cette source va pouvoir contenir d'où viennent les informations relatives à cette naissance.
  • Ajout d'un Objet Multimédia (Add Multimedia Object) - Cet objet va contenir les informations au format binaire concernant cette naissance.
  • Effacer (Delete) - par cette action on va supprimer toutes les informations relatives à cette naissance (on supprime carrément la propriété naissance "BIRT").

Enfin un exemple qui montre comment le menu contextuel peut offrir des possibilités encore plus importantes et cela de manière dynamique en fonction de l'endroit d'où il est lancé. On peut le voir facilement lorsque l'on fait un clic droit afin d'afficher les menus, sur un node (un cadre) dans la visualisation de l'Arbre Généalogique :

  • l'endroit où l'on vient de cliquer n'est pas la représentation d'une propriété, mais une entité elle-même (ici l'Individu I008), aussi le premier sous menu concerne cet Individu.
  • Dans la Visualisation de l'Arbre Généalogique le menu contextuel affiche une action particulière supplémentaire qui est l'ajoût d'un Signet (Add Bookmark).
Les actions présentées dans le sous-menu ouvert ci-dessus, permettent de créer de nouvelles Entités dans le fichier example.ged. Il est fortement recommandé cependant de partir d'informations existantes pour les développer, plutôt que de créer des entités isolées comme des Individus ou des Familles. Pour créer des entités liées ou se rapportant les unes aux autres, utilisez les sous-menus, que ce soit pour les propriétés ou les entités, comme décrit ci-dessus.
