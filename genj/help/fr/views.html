Travailler avec les divers Fen\u00EAtres de Visualisation

Une fois que vous avez ouvert un fichier contenant des données généalogiques au format Gedcom (*.ged), à partir du Centre de Controle vous pouvez ouvrir plusieurs types de visualisation.

Ces visualisations travaillent sur les mêmes données - cela veut dire que

  • les changements effectués dans une fenêtre seront immédiatement effectifs dans les autres fenêtres pour le même fichier,
  • les évènements sélectionnés dans une fenêtre vont être transmis aux autres fenêtres pour le même fichier (*),
  • la fermeture d'un fichier entrainera immédiatement la fermeture de toutes les fenêtres ouvertes sur ce fichier,
  • les menus contextuels lient les actions de tous les types de visualisation sur le même fichier en un endroit

* les changements dans la sélection d'une fenêtre peuvent ne pas provoquer un changement dans une autre si cette dernière ne peut pas suivre la sélection

Typically you will open one instance per type of view - not necessarily all but combinations that help to navigate and understand the underlying data. For example

  • Tree&Table View to browse the existing information
  • Table&Edit View to cleanup, catalog and edit the existing information
  • Report View to run reports on large files

The combination is not limited and you'll soon get a feeling for which combination of views fullfills your needs. The views will remember the position and size of the enclosing windows so position them as required and they will pop-up at the same place next time. You can also open several instances of the same view to check out different angles - in that case they're numbered sequentially.
