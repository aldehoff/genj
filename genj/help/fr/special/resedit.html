ResEdit - L'Éditeur de Ressources

ResEdit est un programme externe à GenJ qui va vous servir à traduire les ressources (les fichiers langages) en d'autres langues (ex. les traduire en français, espagnol, etc....)

De quoi avez-vous besoin pour l'utiliser?

Tout d'abord vous devez créer un fichier de lancement que vous appellerez par exemple resedit (si vous êtes sous Linux), ou resedit.cmd (si vous êtes sous OS/2), etc...

A l'intérieur, mettez les commandes suivantes (l'exemple qui suit concerne Linux, aussi, modifiez les pour que cela corresponde à votre système, bien sûr) :

    #!/bin/sh
    #

    # Les deux lignes qui suivent sont les seules que vous devez changer
    # pour adapter ce script à votre système
    JAVA_HOME=/usr/local/jdk1.2.2
    GENJ_HOME=/home/fm/genj

    echo "/*"
    echo " * Resource Editor startup file for Genj"
    echo " * "
    echo " * If something goes wrong now you probably have an"
    echo " * incomplete CLASSPATH. Either copy swing.jar to"
    echo " * your lib sub-directory or specify its location via"
    echo " * (e.g.) export CLASSPATH=/usr/lib/swing/swing.jar"
    echo " * before starting this script."
    echo " */"

    export CLASSPATH=./lib/resedit.jar:./lib/swing.jar:$CLASSPATH
    export PATH=$JAVA_HOME/bin:$PATH

    cd $GENJ_HOME
    $JAVA_HOME/bin/java com.nm.resedit.Main
Pour lancer votre éditeur de ressources, tout ce que vous avez à faire est d'éxécuter le fichier de démarrage que vous venez de créer, en tapant "resedit".
Aprés avoir cliqué sur le bouton OPEN une fois resedit démarré, choisissez une archive support (ex. genj.jar) mais regardez le N.B. ci-dessous.
Vous pouvez alors, soit traduire les textes existants en chargeant une traduction, soit naviguer à travers les textes manuellement.
Dans cette deuxième hypothèse, vous devez choisir dans la fenêtre de gauche, un chapitre (qui d'ailleurs correspond à un répertoire) pour démarrer la traduction. Ensuite, dans la fenêtre de droite, vous choisissez une ligne à traduire et vous tapez la traduction dans le champ de saisie en bas de l'écran.
Si vous désirez charger une traduction, cliquez sur l'icone "translate" (traduction) dans le menu principal, puis sélectionnez le fichier déjà traduit (il peut n'être traduit que partiellement) que vous voulez éditer/modifier (ex. genj_fr.zip). Une fois sélectionné, décochez l'option "Default local or ..." et choisissez le paramètre local approprié à la langue à laquelle s'applique le fichier (dans mon exemple, il faut choisir "fr"). Une fois cela fait, vous devez cliquer sur le bouton de traduction (translate).
    N.B. A partir de GenJ v. 1.7 et Resedit 0.02, il vaut mieux ouvrir le fichier genj_en.zip plutôt que le genj.jar. Aussi, cliquez d'abord sur le bouton ouvrir (open), puis sélectionnez genj_en.zip. Une fois que c'est fait, cliquez sur le bouton de traduction (translate), et sélectionnez le fichier déjà traduit (il peut n'être traduit que partiellement) que vous voulez éditer/modifier etc...(voir au-dessus).

Aprés avoir traduit les lignes de texte, vous pouvez sauvegarder (SAVE) le résultat dans un fichier compressé au format ZIP ou JAR. Vous devez indiquer à quel paramètre local vous faites référence lorsque vous chargez (LOAD), traduisez (TRANSLATE) ou sauvegardez (SAVE). Cela correspond à une structure de fichier du genre resources_xyz.properties qui est alors utilisé par le logiciel qui charge ces ressources nouvellement traduites. Le nom des fichiers que vous chargez et sauvegardez n'a pas d'importance pour Java - Le paramètre local que vous choisissez est utilisé à l'intérieur de l'archive seulement. Cependant pour que tout le monde s'y retrouve il est plus simple de les identifier avec une dénommination du genre genj_XY.zip (où XY est l'abréviation du paramètre local inséré - ex. genj_fr.zip).
Notes :
- Pour Java 1.1: Vous devez avoir une librairie swing.jar dans votre "classpath".
- Et j'ai également remarqué que pour que Resedit (v. 0.01) fonctionne, les fichiers genj.jar et genj_XY.zip doivent être placés dans le répertoire principal de GenJ, et non dans le sous-répertoire /genj/lib.

Quels fichiers sont disponibles?

Au jour où je traduis cette aide, les fichiers de ressources suivants sont disponibles :
  • Fichier de ressources Allemand : genj_de.zip
  • Fichier de ressources Anglais : genj_en.zip
  • Fichier de ressources Français : genj_fr.zip
  • Fichier de ressources Hollandais : pas encore
  • Fichier de ressources Italien : genj_it.zip
  • Fichier de ressources Japonais : genj_ja.zip
(Les fichiers langage Anglais, Allemand, et Français sont en général mis à jour avec chaque nouvelle version de GenJ)
