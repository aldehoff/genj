I - Questions Générales

  1. Q - Qu'est-ce que c'est que GenJ ?
    R - GenealogyJ (en abrégé GenJ) est un visualiseur et un éditeur de données généalogiques stockées dans le format Gedcom.

  2. Q - Que veut dire le mot Gedcom ?
    R - Lisez les explications ici.

  3. Q - Y a t'il une forum de discussion concernant GenJ ?
    R - Oui, ce forum de discussion existe. Vous pouvez nous y rejoindre ici (forum de discussion@genj.sourceforge.net)
    Il existe également un canal IRC sur le serveur irc.freenode.net canal #genj. D'autres utilisateurs ou/et développeurs peuvent y être présents. Je vous conseille trés vivement d'installer un client IRC sur votre ordinateur et d'y aller.

  4. Q - Comment puis-je être en contact avec les autres utilisateurs de GenJ et les développeurs ? Où puis-je avec avoir de plus amples informations à propos de GenJ ?
    R - Faites un tour sur :
        -  le Site Internet de GenJ,
        -  le Site de développement de GenJ,
        -  et le canal IRC : #genj sur irc.freenode.net.

  5. Q - Comment connaître les modifications apportées chaque nuit dans les versions quotidiennes de développement qu'on appelle les "Nightlies" ? Existe-t-il une liste genre Changelog ?
    R - Les versions quotidiennes se trouvent ici :  Nightlies.
    Attention, ce sont des versions beta, donc, vous les utilisez à vos risques et périls ; n'oubliez pas de faire des sauvegardes de vos données.
    Pour connaître les modifications, il suffit de vous rendre sur ce site. Vous y trouverez le commentaire sur les 20 derniers changements apportés au code.


II - Interface

  1. Q - Je suis assez dérouté par GenJ. Il n'a pas l'air de fonctionner comme les autres logiciels. Quelle approche avoir ?
    R - Le premier conseil que l'on peut donner, est qu'il ne faut pas essayer de l'utiliser comme les autres logiciels. GenJ est basé sur un multifenêtrage, donc il faut mieux utiliser plusieurs fenêtres en même temps (ex : Ouvrir à la fois les fenêtres de l'arbre généalogique, de la table et de l'éditeur). Ensuite il n'y a pas de barre de Menus avec des sous-menus déroulants ; au contraire, GenJ, utilise beaucoup les sous-menus mais au travers du clic droit de votre souris. Essayez, et ceci pratiquement partout, un clic droit vous ouvrira de multiples possibilités.
    Autre élément à prendre en considération  : GenJ est un trés puissant éditeur Gedcom, donc il est vraiment basé sur la structure interne du format Gedcom. Vous devrez vous habituer au langage utilisé, même si c'est vrai, des efforts importants sont faits pour franciser au maximum le logiciel. Les "tags" que l'on pourraient traduire par "étiquettes", voir "propriétés", et les "pointeurs", doivent être des notions qui tout doucement vous deviendront évidentes. Bien sûr, l'éditeur standard sera toujours là pour vous faciliter la tâche, mais l'éditeur gedcom restera toujours, à mon avis, le meilleur outil pour travailler de manière professionnelle sur votre arbre généalogique. N'hésitez pas à passer de l'un à l'autre ; vous apprendrez énormément, et vous construirez votre arbre exactement selon vos désirs.
    Enfin, GenJ demande un peu de temps pour être appréhendé. Ne soyez pas pressé, et donnez vous un peu de temps pour le découvrir, vous ne serez pas déçu.

  2. Q - Comment puis-je faire pour que GenJ utilise un autre langage ?
    R - Téléchargez le pack langage approprié (genj_ll-x.y.zip) pour GenJ (ex : genj_fr-2.3.3.zip pour le fichier français) depuis la Page Principale du Projet et suivez les instructions décrivant son installation. GenJ essaiera d'utiliser le langage de votre système automatiquement. Pour changer intentionnellement de langage (le pack langage doit être installé manuellement) choisissez le menu Visualisations, puis le sous-menu Options. Ensuite sélectionnez le langage de votre choix.
    Vous devrez redémarrer GenJ, pour que cela soit pris en considération.

  3. Q - Je voudrais que GenJ ait un autre look. Comment faire ?
    R - GenJ utilises la boite d'Outils Swing pour présenter les informations à l'utilisateur. Par défaut c'est la configuration de votre système qui sera prise en considération. D'autre part, GenJ vous permet également de changer le look, l'Apparence courante de l'interface.
    Pour cela, téléchargez le pack Apparence (genj_lnf-x.y.zip) depuis la page des projets de GenJ. Supposons que le fichier que vous avez récupéré s'appelle genj_lnf-2.0.zip, vous allez placer le fichier dans la directory d'installation de GenJ (sous un système Unix, ça pourrait être par exemple /home/pierre/GenJ/). Cette directory est celle o? vous avez installé le programme GenJ lui-même. Une fois le fichier dans cette directory, vous le décompactez, et vous allez voir qu'il va se créer une sous directory qui s'appellera "lnf".
    Après avoir redémarré GenJ, choisissez le menu Visualisations et allez dans le sous-menu Options, choisissez la langue que vous voulez, ex : Français. Maintenant, vous pouvez sélectionner un Aspect (et éventuellement un Thème), ex : Skin(arvernesthemepack.zip).

  4. Q - Comment puis-je faire pour qu'un centimètre ait cette taille sur mon écran ?
    R - GenJ utilise le sytème métrique mondial pour de nombreux composants (par ex. dans la Visualisation de l'Arbre Généalogique). Pour que les longueurs de lignes et les tailles des boîtes apparaissent correctement sur votre écran, vous aurez à indiquer à GenJ votre résolution d'écran (qui est donnée par le système mais normalement ne vous occupez pas des paramètres système du moniteur). Pour ajuster explicitement la résolution choisissez le sous menu du menu Aide|A propos et allez à l'onglet Apparence. Ensuite utilisez une règle correspondante à celles montrées pour votre écran.

  5. Q -  Pourquoi les images sont elles affichées différemment ?
    R - Les empreintes ou calques jouent un rôle important dans le rendu des images. En fonction de ce qui doit être affiché dans un cadre, l'espace alloué à une image peut être limité, et donc cette dernière peut être réduite pour laisser de la place au texte. Si il y a assez de place, l'image prendra toute la place disponible jusqu'à concurrence de sa taille réelle (déterminée en fonction de sa taille et de sa résolution).
    Voilà pourquoi plusieurs images peuvent avoir une allure différente.


III - Utilisation

  1. Q -  Pourquoi GenJ n'arrive pas à me charger un fichier gedcom comportant plusieurs dizaines de milliers de personnes ?
    R - Il est possible que vous ayez ce genre de message d'erreur :
          Error while reading line 309866 java.lag.OutOfMemoryError
    Il vous faut rajouter quelque chose au script de lancement.
    Dans le script de lancement, il faut changer la ligne :
    java -jar run.jar
    en :
    java -Xmx512m -Xms32m -jar run.jar
    (donc on voit qu'on rajoute les paramètres -Xmx512m et -Xms32m à la ligne de commande).
    En fonction de votre environnement (Windows, Linux, ...), il est possible que la commande soit écrite :
    javaw -jar run.jar (pour Windows)
    CMD="$JAVA -jar run.jar" (pour Linux)

    Normalement, ça devrait fonctionner sans pb. Si ce n'est pas le cas, trois possibilités : - Augmentez le paramètre -Xmx512m en -Xmx768m.
    - Autre possibilité : ça peut vouloir dire que dans votre fichier gedcom il y a un tag ou un montage de tags qui n'est pas conforme à la norme gedcom. Surprenant, car beaucoup de code a été rajouté pour pallier à ce genre de choses, et charger quand même le fichier, mais bon ...
    - Autre possibilité : Voir le log produit par GenJ et envoyez le à l'équipe de développement.

  2. Q - Comment se fait-il que mon système soit trés lent ?
    R - Vous avez sans doute un certain nombre d'images photos, etc... associées à votre arbre.
    GenJ vous offre la possibilité de n'afficher automatiquement que les images dont la taille est supérieure à xx octets, afin de libérer de la mémoire pour vos programmes. Pour cela, allez dans le menu Options (donc à partir du Centre de Controle, Menu déroulant : Visualisations puis Options), et allez au paramètre "Données Gedcom - Image - Taille maximale pour visualisation directe (kb)". Mettez par exemple "50" ou "80". Vous devriez sentir une trés nette amélioration.
    Bien sûr cela ne vous empéchera pas de pouvoir visualiser n'importe quelle image, mais au moins elle ne sera pas chargée par défaut lors de l'utilisation de GenJ.

  3. Q - J'ai une personne que j'ai déjà créée et je voudrais faire en sorte qu'elle soit considérée comme l'enfant d'une autre personne de mon arbre. La seule option que j'imagine est d'ajouter un nouvel enfant/parent à un enregistrement déjà créé, mais il ne semble pas exister une option pour ajouter une personne déjà créée en tant que parent ou enfant d'une autre personne. Comment puis je faire ça. Même question pour un conjoint ?
    R - Tu cliques sur la personne sur laquelle tu veux ajouter l'enfant. Là tu cliques droit sur cette personne, tu choisis 'Individu xxxx', et dans le menu déroulant, tu prends 'créer un(e) Enfant', et dans le menu qui s'ouvre, tu choisis la personne de ton choix dans le menu déroulant qui t'est proposé.
    De cette façon, aucune personne nouvelle ne sera créée comme enfant de cette personne, mais un lien va être établi entre les deux individus en question.

    La réponse est la même si par exemple tu veux mettre comme mari ou femme une personne que tu as déjà dans ton arbre :
    Tu ouvres l'éditeur de ton choix (ou l'éditeur gedcom ou l'éditeur standard), tu fais un clic droit sur un endroit quelconque et tu choisis l'option "Individu 'Ixxx'" (Ixxxx est le numéro d'identification de la personne sur laquelle tu es). Tu choisis l'option "Créer un(e)(des) Epoux(se)". Un menu va s'afficher dans lequel tu pourras choisir de créer une "Nouvelle Entrée" (ce n'est pas l'option qu'il te faut choisir car par hypothèse tu veux considérer comme conjoint une personne qui existe déjà), où, dans le menu déroulant, choisir la personne que tu veux comme conjoint à celle sur laquelle tu es positionné. Une remarque : la petite icone à gauche représentant un petit bonhomme, te permet de choisir le type de tri dans la liste déroulante (tri par ID, par Nom, par Date de Naissance, par Date de Décès).

  4. Q -  Existe-t'il une solution pour fusionner les lieux, les modifier, afin de faire une base des lieux homogéne ?
    R - Oui,une fois GenJ lancé, vous ouvrez l'éditeur (je préfère l'éditeur Gedcom). Vous vous placez sur la ligne qui contient le lieu que vous voulez modifier (c'est normalement l'étiquette PLAC xxxxx). Vous changez la valeur en ce que vous voulez, et vous verrez sur la ligne d'édition à côté de la petite flèche qui vous permet de dérouler la liste des lieux, une petite case qui apparaîtra. Mettez votre curseur de souris dessus, vous aurez une aide qui s'affichera, disant quelque chose comme "Changer les 18 expressions 'abcdef' par 'lmnopq'". Par exemple si vous aviez sur la ligne PLAC la valeur "La Chaise Dieu" et que vous vouliez remplacer toutes les lignes dans votre fichier gedcom qui contiennent "La Chaise Dieu" en "La Chaise-Dieu (Haute-Loire)" il vous suffit de taper sur une des étiquettes PLAC qui contient "La Chaise Dieu" les mots "La Chaise-Dieu (Haute-Loire)", et GenJ vous proposera de faire la modification partout dans votre fichier en vous indiquant qui plus est le nombre de fois qu'il va le faire, en simplement cochant la case dont il est question ci-dessus.

  5. Q - Comment puis-je présenter des images ou photos dans mon Arbre Généalogique ?
    R - Pour montrer la photo d'une personne dans la vue principale de l'Arbre Généalogique ajoutez une propriété multimédia (OBJE) à vos individus. Allez simplement sur un individu dans la fenêtre d'Edition, choisissez la propriété première INDI, cliquez sur '+' et choisissez OBJE à ajouter. Pointez la sous-propriété FILE vers un fichier dans le gestionnaire de fichiers de votre ordinateur (bouton '>>') et les images seront intégrées par la Visualisation de l'Arbre Généalogique (elle recherche automatiquement l'information INDI:OBJE:FILE).

  6. Q -  Comment puis-je rentrer des informations pour une adoption ?
    R - Il suffit d'ajouter un évènement ADOPTion à l'individu dans la fenêtre d'édition puis d'ajouter une référence à la Famille qui a adopté la personne (par le menu contextuel de l'évènement ADOPtion, toujours en étant focalisé sur cette même personne).

  7. Q -  Je voudrais exporter une partie de mon arbre à partir de mon fichier gedcom, pour n'avoir que les ascendants et descendants d'une personne donnée ?
    R - L'opération se fait en deux temps :
    Tout d'abord, il vous faut ouvrir la fenêtre de visualisation de l'arbre généalogique, et centrer l'arbre sur la personne que vous avez choisie pour être celle dont l'ascendance et la descendance seront seules incluses dans le nouveau fichier gedcom qui doit être créé. Ensuite, une fois que c'est fait, vous allez sauvegarder le résultat sous un nouveau fichier. Donc il faut faire "Fichier - Sauvegarder Sous". Là vous mettez le nom que vous voulez pour ce nouveau fichier (ex: export_dupont_ID148.ged), et avant de faire "Sauvegarder", vous avez sur la droite des onglets dont un s'appelle "Filtrer par Vue". Juste en dessous du titre vous trouverez une case à cocher, avec à côté, un commentaire du genre "( ) xx nodes dans mon_fichier.ged - Arbre Généalogique (1)". (xx sera remplacé chez vous par le nombre de nodes qu'il y a dans le fichier que vous voulez créer, exemple : 245). Cochez la case, et faites "Sauvegarder". Voilà, vous venez de créer un nouveau fichier gedcom qui ne comprend que les personnes que vous vouliez.
