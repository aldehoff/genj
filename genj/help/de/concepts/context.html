Arbeiten mit dem Kontextmenü

Ein wichtiger Punkt bei der Arbeit mit GenJ ist das Verwenden von Kontextmenüs. Ein Rechtsklick (Bei Ihrer Plattform kann eine andere Maus-Aktion erforderlich sein, um das Kontextmenü aufzurufen) auf angezeigte Information in einer Ansicht wird Ihnen zeigen, welche Aktionen Sie ausführen können.

Die Struktur des Kontextmenüs ist immer gleich: an der ersten Stelle erscheint ein zur angeklickten Eigenschaft gehöriges Untermenü. Dann folgen weitere Untermenüs, die den Kontext erweitern.

Im folgendem Beispiel wird das Kontextmenü für die Namens-Eigenschaft angezeigt. Das erste Untermenü zeigt die Eigenschaft und den dazugehörigen Aktionen, das zweite Untermenü zeigt die Instanz (hier ein Individuum), die der Eigenschaft zugeordnet ist, das dritte Untermenü zeigt die Datei (Gedcom), die der Instanz zugeordnet ist.

Das Untermenü für das Individuum 'I008' ermöglicht die folgenden Aktionen:
  • Kind hinzufügen – ein Kind (und wenn nötig eine Familie) für I008
  • Partner hinzufügen – einen Partner (und wenn nötig eine Familie) für I008
  • Geschwister hinzufügen – Geschwister (und wenn nötig Eltern) für I008
  • Bibliothek hinzufügen – ein externe Bibliothek mit Informationen zu I008
  • Multimediaobjekt hinzufügen – ein Objekt mit binären Daten
  • Notiz hinzufügen – eine Notiz zu I008
  • Übermittler hinzufügen – eine Person welche Informationen übermittelt hat zu I008
  • Quelle hinzufügen – die Quelle der Informationen über I008
  • Assoziation hinzufügen – eine Beziehung zu einem anderen Individuum
  • Löschen – Entfernen der Instanz I008

Das nächste Beispiel zeigt ein Kontextmenü für eine Eigenschaft, wobei auch die Aktionen angezeigt werden (erstes Untermenü).

Möliche Aktionen sind:
  • Notiz hinzufügen – diese Notiz kann Daten zur näheren Beschreibung der Geburt enthalten
  • Quelle hinzufügen – diese Quelle kann festhalten, woher die Informationen zur Geburt stammen
  • Multimediaobjekt hinzufügen – dieses Objekt kann binäre Daten bzgl. der Geburt abspeichern
  • Löschen – Informationen über die Geburt (Eigenschaft BIRT) komplett löschen

Zum Schluss ein Beispiel das veranschaulicht, wie der Inhalt von Kontextmenüs dynamisch erstellt wird. Die folgende Logik wird angewendet, wenn das Menü in der Baum-Ansicht bei einem Rechtsklick auf einem Knoten erstellt wird:

  • Der Kontext, auf den geklickt wurde, ist keine Eigenschaft sondern eine Instanz (hier Individuum I008). Somit ist das erste Untermenü für ein Individuum
  • Die Ansicht, in der das Kontextmenü aufgerufen wurde, – Baum-Ansicht – hat einen ansichts-spezifischen Eintrag zum Untermenü hinzugefügt, die Aktion Bookmark hinzufügen.
Die Aktionen, die im geöffneten Untermenü angezeigt werden, erlauben das Hinzufügen einer neuen Instanz in der Datei example.ged. Es ratsam, auf vorhandenen Daten aufzubauen statt neue bzw. alleinstehende Instanzen wie Individuen oder Familien hinzuzufügen. Um verwandte/referenzierte Instanzen zu erzeugen, benutzen Sie das Untermenü für Eigenschaften oder Instanzen wie oben beschrieben.
