Report Erstellung

Ein GenJ Report ist eine Java Klasse welche die Schnittstelle genj.report.Report implementiert. Um eigene Funktionalität zu erstellen, definieren Sie Ihre eigene Klasse in einer Quelldatei and kopieren Sie diese Datei in das Report-Unterverzeichnis .\report (der Dateiname muss mit „Report*“ beginnen).

Lassen Sie uns annehmen, dass wir einen Testreport mit dem Namen "Test1" erstellen wollen. Zuerst erstellen Sie eine Textdatei mit Namen ReportTest1.java im Verzeichnis ./report. Dann kompilieren Sie die Datei, in dem sie das Skript zum Kompilieren, welches sich im gleichen Verzeichnis befindet, starten. Dieses Skript startet javac – den JAVA Compiler. Das Ergebnis ist eine Datei mit Namen ReportTest1.class, welche GenJ laden und ausführen kann, nachdem sie Sie in der Report Ansicht ausgewählt haben.

Lassen Sie uns einen Blick auf das Beispiel ReportTest1.java werfen. Es enthält eine public class-Definition, welche dem Report Auftrag entspricht (implements Report). Ein Teil dieses Auftrages ist das Verhalten für alle folgenden Methoden zu definieren:

·  getName

·  getInfo

·  start

·  usesStandardOut

·  getAuthor

·  isReadOnly

Sehen Sie sich das unten stehende Beispiel an oder die herunterladbare API an, um besser zu verstehen was diese Methoden tun. Die wichtigste Methode ist start – Sie ist der Haupteinstiegspunkt für die Logik des Report: 

 
 
  import genj.gedcom.*;
  import genj.report.*;
  import java.io.*;
 
  /**
   * A report named Foo
   */
  public class ReportFoo implements Report {
 
    /**
     * The name of the report
     */
    public String getName() {
      return "Foo";
    }
 
    /**
     * Information about the report
     */
    public String getInfo() {
      return "This report is oh so good\nPlease try it now!";
    }
 
    /**
     * The main entry point for the report
     */
    public boolean start(ReportBridge bridge, Gedcom gedcom) throws InterruptedException {
 
      bridge.println("Hello World");
 
      return true;
    }
 
    /**
     * Whether this report uses StandardOut or its own windows/dialogs
     */
    public boolean usesStandardOut() {
      return true;
    }
 
    /**
     * The author of this report
     */
    public String getAuthor() {
      return "Nils Meier ";
    }
 
    /**
     * Whether this report will change the Gedcom data it is running on
     */
    public boolean isReadOnly() {
      return false;
    }
  }

Innerhalb der start Methode können auf alle Informationen des Gedcom Objektes zugegriffen werden. Sie müssen die GenJ API (eine HTML Klassenreferenz) herunterladen, um die Möglichkeiten der GenJ Kernklassen zu erforschen. Um sich vertraut zu machen, sollten Sie damit beginnen bestehende Reports zu verändern oder zu erweitern.
