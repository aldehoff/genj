Ansichten - Report Ansicht

Ein Report ist eine Java-Klasse, die eine Schnittstelle zu GenJ implementiert. Reporte sind im Verzeichnis ./reports abzulegen und werden dynamisch von GenJ geladen. Nach Öffnen der Report-Ansicht wird eine Liste der verfügbaren Reporte gezeigt. Durch Auswahl eines Listeneintrages wird rechts der Liste eine Beschreibung des Reportes sowie Informationen zum Autor angezeigt.

Mit den Auswahlknöpfen wird ein Report gestartet, angehalten, nach einer Änderung neu geladen, das Ergebnis gespeichert oder die Ansicht geschlossen, Beachten Sie, dass alle Reporte auf die Gedcom-Datei zugreifen, die zum Zeitpunkt der Auswahl der Report-Ansicht geöffnet war.

Abhängig vom Report-Typ springt die Ansicht nach dem Report-Start zur Ergebnis-Karteikarte. Ein Report kann entweder vom Typ Konsole oder Fenster sein. Der letztgenannte Typ erstellt eine einfache Ausgabe, die innerhalb der Karteikarte dargestellt wird (kann später abgespeichert werde). Beim erstgenannten Typ erscheinen angepasste Dialogfenster, ohne das Ergebnis im Standard-Ausgabeformat zu erstellen.

Um zu erfahren, was man mit Reporten tun kann und wie Sie Ihre eigenen Reporte erstellen können, folgen Sie diesem Link. Das Ausführen von Reporten, die in der LifeLines-Skriptsprache erstellt wurden, ist hier erklärt.
